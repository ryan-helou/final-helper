## 💡 KEY STUDY TIPS BY COURSE

### 🔴 ELEC 275 (HIGHEST PRIORITY – Gets ~210 min/week)

**Week 1–2: Master the Fundamentals**
- Ohm's law: V = IR
- Kirchhoff's voltage law (KVL): Sum of voltages = 0
- Kirchhoff's current law (KCL): Sum of currents = 0
- Nodal analysis: Use KCL at nodes
- Mesh analysis: Use KVL around loops
- Thévenin equivalent circuits
- **Goal:** Perfect understanding of DC circuit analysis

**Week 2–3: AC is Critical**
- Sinusoidal signals: v(t) = V_m sin(ωt + φ)
- Phasors represent sinusoidal signals as complex numbers
- Impedance: Z = R + jX (combines resistance and reactance)
- Power factor: cos(θ) relates real and apparent power
- Real power (P): watts (W)
- Reactive power (Q): volt-amperes reactive (VAR)
- Apparent power (S): volt-amperes (VA)
- **Goal:** Master AC circuit analysis with phasors

**Week 3–4: Power & Transformers**
- Complex power: S = P + jQ
- Power factor correction
- Ideal transformers: V1/V2 = N1/N2 = I2/I1
- Three-phase circuits: 120° phase separation
- Balanced vs unbalanced systems
- **Goal:** Solve power and transformer problems

**Week 4: Magnetic & Electric Machines**
- Magnetic circuits: Reluctance R = ℓ/(μA)
- Energy conversion in machines
- AC motors and generators
- DC motors and generators
- Torque and speed relationships
- **Goal:** Understand machine operation and equations

**Pro Tips:**
1. Create a formula sheet with all equations
2. Practice ~50+ problems (course provides them)
3. Use units throughout: watts, volts, amps, ohms
4. Draw circuits and phasor diagrams
5. Days 28–29: Complete both practice exams timed

### 🟠 COMP 352 (MEDIUM-HIGH – Gets ~140 min/week until April 17)

**Week 1–2: Algorithm Analysis & Basic Data Structures**
- Big O notation: How does runtime grow with input size?
- Time complexity: O(1) < O(log n) < O(n) < O(n²) < O(2ⁿ)
- Space complexity: How much memory is needed?
- Stacks: LIFO (Last In, First Out), push/pop
- Queues: FIFO (First In, First Out), enqueue/dequeue
- Linked lists: Dynamic size, slow access, fast insertion
- Arrays: Fixed size, fast access, slow insertion
- **Goal:** Understand trade-offs between structures

**Week 2–3: Trees & Balanced Structures**
- Binary trees: Each node has ≤2 children
- Tree traversal: Pre-order, in-order, post-order
- Binary search trees (BST): Left < Parent < Right
- Balanced trees: Keep operations O(log n)
- AVL trees: Self-balancing via rotations
- Red-Black trees: Color-based balancing
- Heaps: Complete tree with heap property
- **Goal:** Build and traverse trees, understand balancing

**Week 3: Hashing & Searching**
- Hash tables: Fast lookup O(1) average case
- Hash functions: Distribute keys evenly
- Collision resolution: Chaining, open addressing
- Sorting: Compare different algorithms
- Bubble sort: O(n²), simple
- Merge sort: O(n log n), divide-and-conquer
- Quick sort: O(n log n) average, in-place
- **Goal:** Choose right data structure for problem

**Week 3: Graphs**
- Graph representations: Adjacency list, adjacency matrix
- DFS (Depth-First Search): Explore deep before wide
- BFS (Breadth-First Search): Explore wide before deep
- Shortest path: Dijkstra's algorithm
- Spanning trees: Minimum weight connections
- **Goal:** Understand graph algorithms and applications

**Pro Tips:**
1. Understand Big O – it's foundational to everything
2. Practice writing code by hand or pseudocode
3. Know when to use each data structure
4. Trace through algorithms with examples
5. Study all example solutions in course
6. Day 16: Intensive review before April 17 exam

### 🟠 COMP 335 (MEDIUM-HIGH – Gets ~140 min/week)

**Week 1: FSA & Regular Languages**
- Finite State Automata (FSA): States and transitions
- DFA (Deterministic): One path per input
- NFA (Non-deterministic): Multiple paths allowed
- Regular languages: Languages recognized by DFAs/NFAs
- Regular expressions: Compact notation for patterns
- Kleene star (*): Zero or more repetitions
- Concatenation and union operations
- **Goal:** Design DFAs/NFAs for given languages

**Week 2: Regular Language Theory**
- Closure properties: Union, concatenation, Kleene star
- Complement and intersection properties
- DFA minimization: Merge equivalent states
- Pumping lemma: Prove a language is NOT regular
- Regular grammars: Generate regular languages
- **Goal:** Prove properties of regular languages

**Week 2: Context-Free Languages**
- Context-Free Grammar (CFG): Production rules
- Terminals and non-terminals
- Language generation and recognition
- Simplification: Remove useless symbols
- Chomsky Normal Form: A → BC or A → a
- Greibach Normal Form: A → aα
- **Goal:** Design CFGs for given languages

**Week 3: PDAs & Parsing**
- Push-Down Automata (PDA): FSA + stack
- Deterministic vs non-deterministic PDAs
- Acceptance by empty stack vs final state
- CFL pumping lemma: Prove language is NOT CFL
- Parsing: Check if string is in language
- LL and LR parsing techniques
- **Goal:** Design PDAs for context-free languages

**Week 3–4: Turing Machines & Decidability**
- Turing Machine (TM): Tape, head, state machine
- Multi-tape Turing machines
- Non-deterministic Turing machines
- Turing's thesis: Algorithms = Turing machines
- Decidable languages: Yes/no answer guaranteed
- Undecidable languages: No algorithm exists
- Halting problem: Undecidable
- **Goal:** Understand computation limits

**Pro Tips:**
1. Progression: FSA → CFL → TM (each builds on previous)
2. Proofs are key – practice pumping lemma proofs
3. Use JFLAP tool to visualize and test automata
4. Design automata from language descriptions
5. Connect theory to applications (parsing, compiling)
6. Days 28–29: Complete mixed problem sets
